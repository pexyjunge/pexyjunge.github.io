git相关操作

回退已commit的代码

提交代码时，已将文件commit，但是此时发现有错误，则此时可通过git reset –soft [commitId]回退到指定版本。

如图所示，commit了三次提交了3个文件，当我第三次提交成功后才发现提交的内容有误，则此时可通过如下方法回退版本。

查看提交记录，获取上一次提交的commitId

git log

git reset --soft a2dd0baa2fd86458d9f0eb01a60344180315492e

执行后通过git log 查看提交记录

此时第三次commit记录已经不见了，且第三次提交的代码已进入暂存区，修改后重新提交即可。

若第三次提交后发现第二次提交的代码问题，则找到第一次提交的commitId，通过执行git reset --soft [commitId]，回退commit至第一次提交时的状态，修改代码后再次commit，此时第二三次提交的代码会合并成一个新的commit提交。

回退已push的代码

若代码推送到远程仓库后想回退，则先通过git log查看commitId

再通过git reset --hard 4ffcc10b1ae28f02e50cd7cd863303aa6dba55e6

将分支恢复到push之前。此时通过 git push 命令会报如下错误，因为此时本地分支时落后于远程分支的。通过git push -f即可强制更新，但此时远程分支会被本地分支覆盖，不清楚的话慎用。

若想把已推送到远程的代码回退修改后重新提交

则同”回退已commit的代码”操作一致，最后git push -f即可。

缓存本地未commit代码并从当前分支清除

git stash save "stash备注"

此时会将暂存区的内存缓存起来，最好添加备注，回来看时容易追溯。

取出缓存的代码并清除缓存记录

git stash pop stash@{0}

此时stash{0}缓存的内容恢复到暂存区，stash@{0}缓存被清除。

取出缓存的代码但不清除缓存记录

git stash apply stash@{0}

此时stash{0}缓存的内容恢复到暂存区，stash@{0}缓存仍保留。

删除单个缓存

git stash drop stash@{0}

删除所有缓存

git stash clear

查看缓存记录与当前分支的差异

git stash show stash@{0}
